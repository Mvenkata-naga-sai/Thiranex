# 📝 Task Submission: Interactive To-Do List Application

## 📌 Task Title

Develop an interactive, client-side To-Do List application to master DOM manipulation, event handling, and local data persistence.

---

## 🎯 Objective

The goal of this project is to build a dynamic and interactive To-Do List application using JavaScript, focusing on DOM manipulation, event handling, and persistent data storage using `localStorage`.

---

## 🚀 Key Features Implemented

* ✅ Full **CRUD functionality**

  * Create new tasks
  * Read/display tasks
  * Update/edit existing tasks
  * Delete tasks

* ✅ **Local Storage Integration**

  * Tasks are saved automatically in `window.localStorage`
  * Data persists even after browser reload

* ✅ **Advanced Filtering**

  * View **All Tasks**
  * View **Active Tasks**
  * View **Completed Tasks**

* ✅ **Dynamic DOM Manipulation**

  * Tasks are created and updated dynamically
  * Efficient rendering without page reload

* ✅ **Event Handling**

  * Click events for adding, editing, deleting tasks
  * Delegated event listeners for better performance

---

## 🛠️ Technologies Used

* **HTML5** – Structure of the application
* **CSS3** – Styling and responsive layout
* **JavaScript (ES6)** – Logic, DOM manipulation, and storage

---

## 📱 Responsiveness

The application is designed to work smoothly across:

* 📱 Mobile devices
* 📲 Tablets
* 💻 Desktop screens

---

## ⚙️ Application Workflow

1. User enters a task
2. Task is added dynamically to the list
3. Task is stored in `localStorage`
4. User can:

   * Mark task as completed
   * Edit task
   * Delete task
5. Filters allow viewing tasks based on status

---

## 📌 Expected Outcome

A fully functional, state-driven task management application that:

* Provides seamless user interaction
* Maintains application state
* Retains user data across browser sessions
